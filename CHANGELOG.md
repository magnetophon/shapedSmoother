# Changelog

All notable changes to **shapedSmoother**, newest first.

This file holds the version-to-version history that previously lived in comments
inside `shapedSmoother.dsp`. Comments in the source now describe only the current
code.

## [Unreleased]

- Housekeeping: moved the per-version change narrative out of the
  `shapedSmoother.dsp` comments into this changelog. The in-source comments now
  explain only current behaviour; no code changed.

## [0.3.5]

- Replaced the baked `aucLevelMult` LUT with a degree-8 polynomial fit, now in
  `auc_poly.lib`.
- Why: the LUT was a 257-entry waveform (`auc_waveform.lib`). Because
  `aucLevelMult` is folded into `att`/`rel` — which feed the lookahead, the
  brake, the output delays *and* the `env()` recursion — Faust's box evaluator
  re-expanded the whole 257-entry waveform at every one of those use sites
  (cost = table size × fan-out), pushing `faust -time` evaluation from ~0.3 s to
  ~14 s. A polynomial is ~16 cheap ops, so the same fan-out is free and
  evaluation drops back to ~0.4 s. Runtime cost is unchanged (the multiplier is
  slider-rate, computed once per block).
- The polynomial is least-squares-fit to `aucLevelMultFormula` (still the single
  source of truth in the `.dsp`) and matches it to ~1.8e-4 (~0.0016 dB) ->
  inaudible. Regenerate with `regen_auc_poly.sh shapedSmoother.dsp 8 257`.
- The old table path is kept in the `.dsp` as `table_aucLevelMult` for reference
  (dead code; needs `auc_waveform.lib`, generated by `auc.sh`, re-enabled to
  compile).

## [0.3.4]

- AUC level compensation: the LUT scales `att`/`rel` by `aucLevelMult(shape)` so
  changing the shape no longer changes the average level.
- Consequence: folding the factor into `att`/`rel` makes the reported latency
  shape-dependent (largest at the sharpest shape).

## [0.3.3]

- Fixed the release phase failing to reach 1 for some rel / release-shape
  combinations. The release branch chose its accelerate/decelerate phase via
  `gonnaMakeIt = projected > lookaheadX`, but for a single step `projected`
  equals the target by construction, so the test was decided by rounding. A
  spurious `False` past the derivative peak teleported the phase to the bottom
  branch (~0), permanently losing phase and making the output reach the target
  early (release faster than dialed).
- Replaced the release selector with a direct test — accelerate when before the
  derivative peak or when the target just grew (catch-up) — which matches
  `gonnaMakeIt`'s intended value without the tie. Attack is unchanged (its
  analogous flip pins the phase near 1, benign) and the phase resync is
  unchanged.
- Added `releasePeakPhase` (slider-rate) to drive the new release branch
  selection.

## [0.3.2]

- Removed the dead `remainingCurve` / `new_totalStep` source code. In v0.3 Faust
  dead-code-eliminated it; in v0.3.1 (which calls the new `*S` helpers) it was no
  longer eliminated, adding ~5 transcendentals per sample to the inner loop.
  Removing the source-level dead code eliminates the regression.

## [0.3.1]

- Slider-rate hoisting of per-shape constants. No algebraic restructuring versus
  v0.3: attack/release stay as separate function calls,
  `derivativeBaseAttack`/`derivativeBaseRelease` stay separate, and the four
  `inverseDerivative*Attack`/`*Release` stay separate. The only change is that
  quantities depending solely on the shape sliders (which Faust failed to
  auto-lift through `select2(releasing, ...)`) are computed per-shape and
  selected scalar-only inside `env()`.
- Specifically hoisted:
  - `attackShape`, `releaseShape` (`shapeMap` of the slider)
  - `attackInvCurveScale`, `releaseInvCurveScale` (`1/curveScale`)
  - `attackZero`, `releaseZero` (`cheapCurveBase(c, 0)`)
  - `attackMaxDerivBase`, `releaseMaxDerivBase` (`maxDerivativeBaseAttack`)
- Added the `*S` helper variants — `cheapCurveReleaseS`/`cheapCurveAttackS` and
  `derivativeReleaseS`/`derivativeAttackS` — which take precomputed
  `1/curveScale` (and `cheapCurveBase(c,0)`) instead of recomputing them per
  sample. Same result as the non-`S` versions.
- Everything else is byte-identical to v0.3.

## [0.3] and earlier

Baseline that the 0.3.1–0.3.5 entries above are measured against. Detailed
history for 0.3 and earlier was not recorded in the source.
